Roteiro de teste

1

Criar a história no Jira

  1. Abra matheuscv.atlassian.net → projeto SCRUM
  2. Crie um novo Issue do tipo Story (ou Task) com qualquer título
    Ex: "Teste Fase 1 — Squad Agêntica"
  3. A história nasce em Backlog — nada acontece ainda (comportamento correto)
2

Disparar o fluxo automático

No board do Jira, mova o card de Backlog → A Refinar.

Isso dispara o webhook para o serviço em Render, que enfileira o job no BullMQ e inicia o fluxo automático.

3

Fluxo automático esperado (~10–30 segundos)

Backlog
  └─ [você move] → A Refinar
       └─ [orquestrador] → Em Refinamento    ← moveCardTo automático
             └─ [Agente PO] → Aguardando Aceite PRD  ← gera PRD.md, commita, comenta

O card para em Aguardando Aceite PRD e não avança mais automaticamente — gate humano 1/5.

4

Onde ver os resultados

O quê Onde verificar
Logs em tempo real Render → agentic-squad-heavy → Logs
Filtre pelo jiraKey da história (ex: SCRUM-42)
Card movido Jira board — card deve estar em Aguardando Aceite PRD
Comentário do bot Abra o card no Jira → aba Comments
Mensagem: "🤖 Agente PO (stub) concluiu a geração do PRD"
PRD.md no GitHub github.com/matheuscv/agentic-squad_heavy
Pasta SCRUM-NNN/PRD.md
Artifact no banco Supabase → Table Editor → tabela artifacts
Filtro: file_path = 'SCRUM-NNN/PRD.md'
Coluna github_commit_sha deve estar preenchida
Execução do agente Supabase → tabela agent_runs
Filtro: agent_type = 'po' e status = 'completed'
5

Checklist de aceite

  • Card em Aguardando Aceite PRD no Jira
  • Comentário do bot visível no card (🤖 Agente PO (stub)…)
  • Arquivo SCRUM-NNN/PRD.md criado no repositório GitHub
  • Linha em artifacts com github_commit_sha preenchido
  • Linha em agent_runs com status = 'completed' e duration_ms registrado
  • Nenhum log de erro crítico no Render (warnings de comentário Jira são tolerados)
